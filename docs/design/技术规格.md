## 1. 愿景与约束

- **目标**：纯本地 Android App，用间隔重复（FSRS）+ 多维卡型 + 认知科学编码仪式，帮考研学生把数学公式记牢。
- **核心约束**：
  - **核心功能零网络依赖**：OCR 仅作*可选增强*，经接口注入；离线可用纸笔自评。
  - **内核非大模型驱动**：记忆调度 / 判定 / 卡型路由全是**确定性算法**（FSRS / 状态机），可解释可复现。AI 只用在**输入识别**与**内容标注**两端。
  - **高标准 UI**：严格 Material 3 + 深色模式。
  - **高质量渲染**：LaTeX 用 KaTeX 离线渲染，禁止 JLatexMath。

## 2. 技术栈 / 架构

见 [`架构总览.md`](架构总览.md) §二、§三。要点：Kotlin + Compose M3 + Room(v11) + KaTeX/WebView + Coroutines/Flow + WorkManager + DataStore(Tink)；严格 **MVVM + UDF**，UI 状态封 `UiState` 经 `StateFlow` 暴露。

## 3. 架构铁律（不可违反）

- 禁止在 Composable 里直接操作数据库 / 解析复杂 JSON——这些在 ViewModel 完成。
- UI 状态必须封在 `UiState`，经 ViewModel `StateFlow` 暴露。
- OCR 必须经 `MathOcrRecognizer` 接口注入；UI 层不感知任何具体识别 SDK。
- 渲染统一 KaTeX + WebView，**禁止引入 JLatexMath**。
- 所有用户文字输入入口必须**有理由**——默认走 chip / 按钮 / 多选（错题录入、cloze 填空等均 chip/数字键盘）。
- 数一/数二/数三过滤必须经 `kaoyanSubject` 守卫；Scene 行为经 `UseScene` enum 分支。
- 6 子卡分别落库，独立 FSRS；母卡 `study_states` 已退役，勿再写。

## 5. 手写识别（OCR）· 可选增强

- **依赖倒置**：Domain 层 `MathOcrRecognizer` 接口（笔画 → Top-N LaTeX 候选）。UI 层完全解耦。
- **实装**：SimpleTex（国内，标准 + Turbo 双端点）+ Mathpix（海外，付费）；`RecognizerRegistry` 选择、`RecognizerPreference`（DataStore + Tink 加密）存 Key。
- **⚠️ ML Kit Digital Ink 已被拒**：实测不支持数学公式（`fromLanguageTag("zxx-Zmth")` 返回 null），勿再考虑。
- **输入方式**：`InputMode` 二选一——手写识别 / 纸笔自评（设置页可切）。
- **反馈闭环**：识别失败样本落 `ocr_feedback`，SAF 导出 JSON。

## 6. 静态内容（公式库）
