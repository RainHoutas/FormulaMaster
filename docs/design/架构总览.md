## 一、一句话架构

一个 **纯本地 Android App**（Kotlin + Compose M3 + Room），把每条数学公式拆成 **6 个记忆维度（子卡）**，各自跑独立的 **FSRS 间隔重复**，由 **复习路由器** 编排轮转，配 **七步编码仪式**（首次学习）与 **错题反向链路**（把做错的公式提前拽回复习）。核心调度**全确定性、不依赖大模型**；AI 只用在手写识别与内容标注两端。

## 二、技术栈（现状）

| 层 | 选型 |
|----|------|
| 语言 / UI | Kotlin + Jetpack Compose（Material 3） |
| 架构 | 严格 MVVM + UDF；UI 状态封装 `UiState`，经 ViewModel 的 `StateFlow` 暴露 |
| 持久化 | Room（SQLite，**当前 DB version = 11**）+ DataStore（偏好；敏感项 Tink AES256-GCM + Android Keystore） |
| 异步 | Coroutines + Flow |
| 公式渲染 | KaTeX 离线包 + WebView（`MathFormulaView` + WebView 复用池）；**禁止 JLatexMath** |
| 后台 | WorkManager（`DailyReminderWorker` 每日复习提醒） |
| 手写识别 | 云端 API（SimpleTex 国内 / Mathpix 海外），经 `MathOcrRecognizer` 接口注入；**ML Kit 已被拒**（不支持数学公式） |

## 四、核心机制（现状，⚠️ 与原型期差异大）

### 1. 场景 / 子科目过滤
- `UseScene`：`KaoyanMath`（默认，唯一实装）/ `Gaokao` / `SelfStudy`（仅占位）。
- `KaoyanSubject`：数一 / 数二 / 数三；进 App 后 `FormulaRepository.observeFormulasFor(subject)` 按 `formula_subject_map` JOIN 过滤，用户只看到本子科目公式。

### 2. 六类子卡 + 子卡级 FSRS（**母卡已退役**）
- `CardType`：C1 识别 / C2 加权 cloze / C3 条件先行 / C4 推导自评 / C5 易混辨析 / C6 题型反查。
- 每条公式对应 **6 张 `SubCardStateEntity`**（复合主键 `formulaId + cardType`），各自独立 `stability / difficulty / lapses / nextReviewTime`。
- ⚠️ **母卡 `StudyStateEntity`（study_states）在 Sprint 2 Task 2.6 已退役**——仅保留表结构兼容，所有 FSRS 读写迁到子卡；整体进度（learningState / lapses 等）由 `SubCardAggregator` 从 6 子卡**派生**。
  > 原型期文档说 `study_states` 是"最核心状态表"——**那是过时信息，勿信**。
- FSRS 由 `ReviewScheduler` 计算，`nextReviewTime` 截断到每日刷新整点（`AppPreference` 可配）。

### 3. 复习路由器（`domain/ReviewRouter.kt`，纯状态机）
- 跨公式**轮转** + 答错**粘卡** + 连续评 1 入**加强卡**集合（回考）+ **默写**（hint 递进，连错 3 次 → `blocked_formulas`）。
- 会话由 `RouterReviewViewModel.buildSessionInputs` 组织 due 子卡（强标记优先 → examWeight 降序）；**同日续接**：会话进度持久化到 `review_session_progress`（跨日由 sessionDate 决定丢弃/续接）。
- ⚠️ **未实装卡型在会话构建期剔除**：C5（延后 Sprint 4）恒剔除；C6 对同章 <2 公式的情况剔除（凑不齐候选池会回落"看答案"）。

### 4. 七步编码仪式（`FormulaLearnRitualScreen`，首次学习）
条件+用途先行 → 拆块讲解 → 推导链 → 临摹（`TracingCanvas`）→ worked example×2 → 最小填空预热 → **巩固迷你卡序列**（C1+C2+C3 混合，全过才结业）。结业初始化 6 子卡 `S=1.0 / nextReview=次日刷新整点`。

### 5. 错题反向链路
- 错题本 UI（Memory 右下角 FAB → `ErrorBookScreen`）：全 chip / 数字键盘录入 + 公式多选池（未学灰显 + 跳学习）。
- `ErrorReportProcessor.process`：录入即把所选公式 6 子卡 `S ← MAX(S×0.5, 0.5)` + 推次日 + `lapses+1`，并**快照惩罚前原值**（`penaltySnapshotJson`）。
- 删除可选「恢复计划」：逐子卡 best-effort 还原——仅还原**录入后未被真实复习触碰**的子卡（`lastReviewTime ≤ createdAt`），复习过的保留。策略持久化（每次询问 / 仅删记录 / 恢复计划）。

### 6. 顽固节点（Leech）· 全 App 统一判定
- `LeechDetector.isLeech(lapses, recentErrorMarks)` = `lapses ≥ 4` **或** `近 7 日被错题反向标记 ≥ 2 次`（`ErrorMarkTally` 计数）。
- 唯一判定出处，全 App 一致：记忆卡红底 / 详情页横幅 / 复习卡顶「🔥 顽固」chip / 测试答错加强震动。

### 7. 手写识别（可选增强）
- `MathOcrRecognizer` 接口注入；`InputMode` 二选一：手写识别 / 纸笔自评。核心功能离线可用。
