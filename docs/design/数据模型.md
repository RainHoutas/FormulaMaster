# Formula Master · 数据模型（当前状态权威）

> **本文档反映现状**（DB **version = 11**，9 张表；学习流程重构阶段 Sprint 1-3 完成）。
> 字段以 `data/local/entity/*.kt` 源码为最终准。本文给结构与语义全景，供新对话快速对接。
> ⚠️ 原型期的《核心数据库和算法设计》（本文件前身）把 `study_states` 当核心表——**已过时**，现以子卡 `sub_card_states` 为 FSRS 真相源。

---

## 一、设计原则

- **静态资产 / 动态状态严格分离**：公式内容（`formulas`）随 App 打包预置、只读；用户学习状态独立成表，公式更新绝不覆盖进度。
- **子卡为真相源**：FSRS 调度落在 6 张子卡（`sub_card_states`），母卡 `study_states` 退役仅留兼容。
- **迁移策略**：`fallbackToDestructiveMigration(dropAllTables=true)`——升级即重置，公式由 `assets/formulas.json` 重新预加载，FSRS 进度量小可重新激活，避免手写 Migration 的工程开销（打磨/重构阶段允许）。

## 三、关键表 schema

### `formulas`（FormulaEntity，18 字段）
静态公式库，`assets/formulas.json` 预加载。

- `formulaId`(PK) / `subject`(高数/线代/概率论) / `chapter` / `title` / `latexCode`
- `clozeData`：加权填空 JSON（`ClozeItem`：`index / placeholder / options / weight / mustBlank`）
- `derivationSteps`：**对象数组** `[{latex, note}, ...]`（⚠️ 原型期是纯字符串数组，已重写）
- `tags` / `difficultyLevel`
- **重构新增**：`purpose`（一句话用途）/ `preconditions`(JSON) / `parents` / `siblings` / `confusableWith`(JSON) / `typicalProblems`(JSON 纯文本题面数组) / `commonErrors`(JSON) / `mnemonic`(可空口诀) / `examWeight`(默认3) / `scene`(默认 KaoyanMath)

### `sub_card_states`（SubCardStateEntity）· FSRS 真相源
复合主键 `(formulaId, cardType)`；一条公式 6 行（C1~C6，`cardType` 存 `CardType.code` 字符串）。

- `stability`(S,天) / `difficulty`(D,[1,5]) / `lastReviewTime` / `nextReviewTime`(截断到刷新整点)
- `totalReviews` / `lapses`(遗忘计数) / `consecutiveGoodReviews`(连续 R≥3) / `isReinforced`(强标记：加强卡回考再失败)
- 外键级联 → `formulas`。整体进度由 `SubCardAggregator` 派生：`learningState = MIN(S)<1→学习中 / AVG>30→已掌握 / 其余→复习中`；`lapses=SUM`；`nextReviewTime=MIN`。

### `error_reports`（ErrorReportEntity）· 错题反向
- `id`(PK,auto) / `createdAt` / `subject` / `chapter` / `sourceType` / `sourceTag`（受限数字编码）
- `wrongFormulaIdsJson`：错位公式 id 数组
- `penaltySnapshotJson`：录入时、施加惩罚**前**所选公式各子卡 `(cardType, stability, nextReviewTime, lapses)` 快照，供删除「恢复计划」逐子卡 best-effort 还原（Sprint 3 Task 3.3 新增，DB v10→v11）
- `note`：预留可空，UI 暂不开放

### `study_states`（StudyStateEntity）· ⚠️ 已退役
Sprint 2 Task 2.6 起 FSRS 读写全部迁到子卡。表结构保留仅为兼容老库；`StudyStateDao` 标 `@Deprecated`。**新逻辑一律走 `sub_card_states` + `SubCardAggregator`，勿再写母卡。**

### 其余表（字段以源码为准）
- `formula_subject_map`：复合主键 `(formulaId, subjectType)`，外键级联；`FormulaDao.observeByKaoyanSubject` 用它 JOIN 过滤。
- `blocked_formulas`：公式级 blocked 状态；默写连错 3 次写入，UI 在默写界面顶部红条提醒；CASCADE 删除。
- `review_session_progress`：单行表，存 `sessionDateMs + formulaContextsJson + currentCursor`；同日续接、跨日丢弃；序列化经 `ReviewSessionProgressCodec`（扁平 DTO，未知 CardType code 静默剔除，损坏 JSON 返回 null 不闪退）。
- `review_logs`：每次评分落一条，支撑热力图/溯源。
- `ocr_feedback`：识别失败样本（笔画 + 候选 + 用户标错部件），SAF JSON 导出。

## 四、FSRS 调度要点（`domain/ReviewScheduler.kt`）

- 评分 1(Again) / 2(Hard) / 3(Good) / 4(Easy)；R=1 视为遗忘 → `lapses+1` + S 重置。
- 稳定性 S 单位为天；`nextReviewTime` 统一截断到用户配置的每日刷新整点（时区安全）。
- 错题反向惩罚：`S ← MAX(S×0.5, 0.5)` + 推次日（见 `ErrorReportProcessor`）。
- 冲刺模式：距考试 ≤ 阈值时批量对已掌握子卡 `S/2` + 拉回复习池（`SprintModeManager.applyIfNeededSubCards`）。
