# Formula Master：考研数学公式学习—复习—考试全流程设计研究报告

> 一份基于认知科学论文、考研名师与高分学员实战经验、同类记忆/学习类产品对比的可落地产品设计依据。
> 适用产品：面向中国"考研数学（高数 / 线代 / 概率论与数理统计）"的 native Android 公式记忆与复习应用 *Formula Master*。

---

## TL;DR（三条核心结论）

1. **不要把"公式记忆"当成"单词记忆"。** 单词卡的核心是"形‑音‑义"的多通道关联，而数学公式是**条件‑结构‑推导‑题型四位一体的程序性+陈述性混合知识**。"不背单词"那一套（选词义→拼写→听力→例句）只能借鉴其**渐进难度阶梯、间隔重复触发、沉浸刷词、激励机制**，而**核心交互必须替换**为：前置条件首读、结构化拆块、推导追溯、题型反查、易混辨析、变式默写。直接照搬选词义/拼写会落入 SuperMemo "Rule 1：不理解就别背"和 Bjork "Undesirable Difficulty"陷阱。

2. **顶层框架建议为五阶段闭环：编码（Encoding）→ 巩固（Consolidation）→ 提取（Retrieval）→ 辨析（Discrimination）→ 迁移（Transfer）**，对应用户使用的"一轮系统学习—二轮强化—三轮真题—冲刺"四个时段，由 FSRS（基于 Difficulty‑Stability‑Retrievability 三组件的现代间隔重复算法）+ 题型反向触发 + 错题驱动复习池 三条调度通道协同。每个公式作为一个**知识对象（Formula Object）**，自动派生 6 类卡片（识别 / 拆块填空 / 条件 / 推导 / 辨析 / 题型应用），并以"公式族图谱（Formula Family Graph）"作为顶层导航。

3. **可落地的关键交互创新**（区别于 Anki/不背单词/Brilliant 的差异化）：
   (a) **拆块临摹**：泰勒、积化和差等长公式自动拆为"通用结构 + 阶乘特征 + 符号规律 + 收敛条件"四块逐块手写；
   (b) **条件先行卡**：每张正式卡片在公式本体出现前 2 秒强制展示"使用前置条件"；
   (c) **5 秒推导回放**：对同根公式族（诱导公式、和差化积、半角倍角）默认不死记，复习时要求用户从"基本公式 + 一句口诀（奇变偶不变）"现场推导；
   (d) **题型↔公式双向索引**：除了"看公式回忆题型"，还要训练"看到题面特征反向激活该用哪个公式"；
   (e) **错题→公式反向链路**：用户在错题本中标记"哪条公式用错"后，该公式的 retrievability 在 FSRS 中被强制下调，自动进入次日复习池。

本研究在三类来源间均衡取证（每类约 1/3 权重）：

**A. 认知科学与教育心理学论文/教科书**

- Ye et al. (墨墨背单词 KDD 2022, IEEE TKDE 2023) 的 DHP 记忆模型与 SSP‑MMC 调度算法

- "知能行"App（考研数学综测，类似背单词 App 的算法 + 数学题）的实践口碑

## 二、为什么"公式 ≠ 单词"——必须重新设计的理论依据

| 维度 | 单词记忆 | 数学公式记忆 |
|---|---|---|
| 知识类型 | 主要是**陈述性**（form↔meaning）| **陈述性 + 程序性混合**（公式本体 + 适用步骤）|
| 记忆单元 | 大致原子化（每个词独立）| **族化、可推导**（一个根公式 → 一组族）|
| 错误代价 | 写错单词→沟通障碍 | 用错条件→整道题 0 分（武忠祥"标准零分"）|
| 上下文 | 例句即可建立 retrieval cue | 必须绑定**题型** + **条件**才能正确激活 |
| 拆解粒度 | 词根词缀拆解 | **结构拆块**（如泰勒：通用结构 + 阶乘 + 符号 + 余项）|
| 间隔重复友好度 | 高（SM‑2 即可）| 中（Donovan & Radosevich 元分析：spacing 效应在高复杂度任务上变弱；FSRS 等更现代算法更合适）|
| 交错效益 | 中等 | **极高**（Rohrer 2015：72% vs 38%）|

**结论：** 公式记忆必须以"理解 → 程序性熟练 → 条件警觉 → 题型识别"为内核；机械抽认卡只是辅助手段，且必须配合 cloze、worked example、interleaving、retrieval‑with‑feedback。SuperMemo Rule 1（"Do not learn if you do not understand"）和 Bjork "undesirable difficulty"（学习者无背景知识时难度反而有害）警告我们：上来就让用户默写一个他没理解的复杂公式，不仅效率低，还会污染 FSRS 数据。

## 三、顶层框架：五阶段闭环 × 三轮使用场景

### 3.1 五阶段认知模型（每个公式都要走完）

```
[1 编码 Encoding]
    ↓ 拆块、可视化、worked example、推导动画、条件高亮
[2 巩固 Consolidation]
    ↓ 同日内多次短间隔复习（学习步：10m/30m/1d）+ 拼写式临摹 + 即时反馈
[3 提取 Retrieval]
    ↓ Cloze 填空 → 选项辨析 → 限时默写（递进式 desirable difficulty）
[4 辨析 Discrimination]
    ↓ 易混公式 A/B 选择、条件辨析题、错题反向激活
[5 迁移 Transfer]
    ↓ 题型→公式反向激活、变式题应用、限时模拟卷
```

每阶段对应的**主要科学原理**：

| 阶段 | 主要原理 | 主要交互 |
|---|---|---|
| 编码 | Worked example effect (Sweller)；Chunking (Chase‑Simon)；Elaborative encoding (Craik‑Lockhart)| 拆块讲解、推导动画、条件首读卡、费曼式类比 |
| 巩固 | Spacing effect; Forward effect of testing | 同日短间隔；学习后立即一次"识别+条件回填" |
| 提取 | Testing effect (Roediger‑Karpicke); Retrieval strength (Bjork)| Cloze、选项、限时默写 |
| 辨析 | Interleaving (Rohrer); Combat interference (SuperMemo Rule 11)| A/B 辨析卡、易混专项 |
| 迁移 | Transfer of training (ACT‑R); Procedural fluency (NCTM)| 题型识别题、变式应用、错题回路 |

### 3.2 三轮使用场景的策略切换

| 阶段 | 用户状态 | App 默认推送策略 |
|---|---|---|
| **一轮（基础，3‑7 月）** | 跟着汤家凤/武忠祥课程系统过教材 | 以**编码 + 巩固**为主：每日新公式 6‑10 个；每个公式必须看完"拆块讲解 + 推导动画 + 条件高亮"才解锁；当日学完只考"识别 + 条件单填空"；FSRS desired retention 设 90%（容错高）。**不交错** chapter 边界以内的同族公式（block 内部用相似结构帮助形成 chunk）。|
| **二轮（强化，7‑10 月）** | 全书 + 660/1000/1800 题集 | 以**提取 + 辨析**为主：开启**章内交错** + **跨章节弱交错**；提升 Cloze 多挖空比例；引入易混 A/B 辨析；题型→公式反向卡上线。FSRS 调到 88%。|
| **三轮（真题，10‑11 月）** | 历年真题 + 模拟卷 | 以**迁移**为主：限时默写、整套公式串复习（每日 10 分钟"今日真题用过的所有公式快闪"）；错题驱动反向召回；增加限时挑战（响应速度计入掌握度）。|
| **冲刺（12 月‑考前 1 周）** | 模考、查漏补缺 | 切到**保持模式**：FSRS 调到 92‑95%（高保留），关闭新公式；"易遗忘 Top 20"专属面板；考前 3 天进入"晨间公式闪回"5 分钟模式；考前一周回归基本概念、公式、定理（与考研帮、中国教育在线主流建议一致）。|

## 四、学习阶段（Encoding）的具体交互设计

### 4.1 公式首次呈现的"九栏数据结构"

每个公式作为 **Formula Object**，必含字段（直接对应数据库 schema）：

```
formula_id, latex, name (中/英)
preconditions[]      # 前置条件（如：f 在 x0 处 n 阶可导）
domain_constraints[] # 定义域/收敛域
parent_formulas[]    # 推导上游（5 秒推导链）
sibling_formulas[]   # 同族（如 sin/cos 的诱导公式）
confusable_with[]    # 易混对（dx vs Δx, 二重 vs 二次）
chunks[]             # 拆块单元（见 4.2）
key_blanks[]         # 关键挖空点（见 5.2）
typical_problems[]   # 绑定题型 ID
common_errors[]      # 易错点（"忘记验证条件"等）
mnemonic             # 口诀/类比/费曼式解释
chapter, exam_weight # 章节、考频
```

### 4.2 拆块（Chunking）规则——以泰勒公式为例

按 Chase‑Simon chunking theory，专家记忆是高层 template + 槽位填充。把"f(x)=Σ f^(n)(a)/n! · (x‑a)^n + R_n"拆为：

| 块 | 内容 | 记忆要点 |
|---|---|---|
| 块 1：通用结构 | 求和符号 Σ | "导数除阶乘乘幂"——程序性骨架 |
| 块 2：阶乘特征 | 1/n! | 每加一阶就除以下一个整数 |
| 块 3：幂次规律 | (x‑a)^n | 与导数阶数同步递增 |
| 块 4：符号规律 | sin/cos/(1+x)^α 各自规律 | 用"sinx=x 起步、像数字 3"等图形助记（B 站、知乎广泛流传）|
| 块 5：余项 | Peano / Lagrange | 单独考"哪类题用哪类余项"|
| 块 6：收敛/适用条件 | 在 x=a 邻域 | **首屏强制展示** |

每个块都是一张独立 Cloze 子卡（见 §6.2 卡片矩阵），但调度时**先 block 同族内复习**（让大脑形成 template），二轮再交错。

### 4.3 临摹之外应有的环节（推荐 5 步学习仪式）

参考"不背单词"四次连续答对掌握 + Anki 学习步 + Brilliant step‑by‑step：

```
Step 1  条件先行卡（2 秒强制展示） — 先看适用条件，再看公式
Step 2  分块讲解 + 推导动画         — 用类似 3Blue1Brown 风格的逐帧动画展示从父公式推出
Step 3  对照临摹（手写 SDK）        — 用 MyScript / Selvy Pen / Mathpix Digital Ink，
                                     边写边识别，错的笔画即时高亮
Step 4  Worked Example × 2          — 立刻看一道用此公式的简单例题（Sweller 效应：novice 阶段 worked example > 立刻做题）
Step 5  最小填空预热                — 仅挖一处关键点，确认理解 → 才进入 FSRS 队列
```

### 4.4 让用户"理解"而非"机械"记忆的三大手段

1. **推导链可视化**：每个公式有"展开推导"按钮，点击后用 3‑5 帧 LaTeX 渐变展示从父公式 → 子公式（参考 Khan Academy 视频风格）。
2. **费曼式 30 字大白话**：每个公式额外存一段用户/编辑提供的"用最朴素话解释这个公式干嘛"，读 1 遍并提示用户"试着合上 App 给同学讲一遍"——这是费曼学习法 4 步在 App 里的轻量化落地。
3. **二级结论提示**：知乎反复强调"二级结论不要死记，要在理解后顺手记一下"——App 提供"扩展二级结论卡"开关，默认关闭，避免初期信息过载（SuperMemo Rule 4：minimum information principle）。

### 4.5 公式族图谱（Formula Family Graph）

参考思维导图工具（boardmix、GitMind、Freeplane）和墨墨"派生词汇树"思想。每个用户首次进入一个章节时：

- 自动生成本章公式族图（节点 = 公式，边 = "由…推出"或"易混于…"）
- 学习过的公式自动点亮（与 Khan Academy mastery 颜色梯度类似：未学/Familiar/Proficient/Mastered）
- 用户可点击图中任意节点跳转到该公式卡
- **顶层导航**：图谱不是装饰，而是替代传统"目录"作为主入口之一

## 五、复习阶段（Retrieval + Discrimination）的具体交互设计

### 5.1 难度梯度（递进式 desirable difficulty）

按 Bjork 理论与 Anki Cloze 经验，三档梯度：

```
Easy   选择题（4 选 1）        — 给提示，激活长期记忆
Medium 关键点 Cloze 填空        — 单挖空、多挖空可调
Hard   全公式默写（手写识别）    — 限时 + 必须先写条件
```

App 根据 FSRS 的 retrievability 自适应选择难度：R<0.7 给 Easy；R∈[0.7,0.9) 给 Medium；R≥0.9 给 Hard（与 Brilliant.org 自适应思路一致）。

### 5.2 挖空算法（核心创新点）

**哪些位置应该挖空？** 基于 SuperMemo Rule "minimum information"、Cloze 研究和 Bjork 提取强度：

1. **必挖位**（强制）：
   - 适用条件中的关键限定词（如"f 在 [a,b] **连续** 且在 (a,b) **可导**"中的两个红字）
   - 阶乘/幂次/符号交替规律的"位置代码"（如 (-1)^n、n!）
   - 易混点（如"是 2π 还是 π"）

2. **概率挖位**（动态）：
   - 用户最近 3 次该公式做错的位置 → 挖空概率 ×3
   - 该位置在历史考研真题中曾是命题点（题库标记） → 挖空概率 ×2

3. **避免挖位**：变量名、明显的结构性符号（Σ、∫）、定语连接词

**单挖 vs 多挖：** 一轮单挖 → 二轮 2‑3 挖 → 三轮全挖（=默写）。这是 Bjork 的"逐步施加 desirable difficulty"。

**算法：** 每个公式预先由编辑标注 5‑8 个候选挖空位置 + 优先级权重；每次复习时根据 FSRS 状态 + 用户历史错误抽样 1‑N 个挖空。**不固定**——保证 Bjork variability。

### 5.3 易混公式辨析卡（A/B 题）

题型：给定一个题面或一个表达式，问"应该用 A 公式还是 B 公式？为什么？"

例：
- "二重积分极坐标变换 vs 三重积分柱坐标变换"
- "拉格朗日中值定理 vs 柯西中值定理"
- "正态分布 → 标准化 vs 卡方分布定义"

辨析卡是独立卡片类型，FSRS 单独调度（不混在原公式卡里），这是借鉴 SuperMemo Rule 11"combat interference"。

### 5.4 题型→公式反向激活（关键差异化）

UI 流程：
1. App 推送一道**题面（不让做完整题）**
2. 让用户从公式池中点选 / 输入"这道题最关键应该用哪个公式"
3. 提示用户"用关键公式写一行核心推导"

数据来源：每个公式预先绑定 N 道典型题；强化阶段每次会话 30% 抽到反向卡。这是 ACT‑R cognitive tutor"识别该用哪条 production rule"的轻量化实现。

### 5.5 在复习中触发推导回顾（5 秒推导挑战）

对标记为"推导优先派"的公式（诱导公式、和差化积、积化和差、半角倍角等），复习时：
- **不直接挖空公式本体**
- 而是显示一个 timer（5 秒），让用户在心里/草稿上"从父公式 + 口诀推一遍"
- 然后展示推导链 + 结果，让用户自评"推出来了 / 看了才会 / 完全不会"
- 自评结果作为 FSRS 反馈

这正是中国数学高分学员实战经验（知乎多篇）的产品化：**高手不死背，背"基本 + 5 秒推导路径"**。

## 六、考试阶段（Transfer）的具体交互设计

### 6.1 三档考试模式

| 模式 | 设置 | 用途 |
|---|---|---|
| 全默写 | 公式名 → 用户手写完整公式 + 适用条件，无时间限制 | 二轮收尾 |
| 限时默写 | 同上但 60s/120s 限时 | 三轮 |
| 公式模拟卷 | 每章抽 10‑20 个公式，混合识别/默写/题型反查 | 冲刺 |

### 6.2 错误回流到复习池

错题处理流程（与"错题→公式"反向链路结合）：

```
默写错误 → 自动定位错在哪一块 → 该块的 cloze 子卡 retrievability ↓
       → 如果错的是"条件"而非"本体" → 标记为"条件型遗忘"，专项条件辨析卡 +1
       → 加入"今日错题"小黑屋，第二天必出
       → 7 日内仍错 ≥2 次 → 加入"长期错题观察池"（参考知能行/墨墨"小黑屋"）
```

### 6.3 掌握程度评估（多维度）

不只是答对率，参考 Khan Academy 4 级 + ACT‑R skill strength + 数学考试现实：

| 维度 | 测量 | 权重 |
|---|---|---|
| 正确率 | 近 N 次正确次数 / N | 35% |
| 反应速度 | 默写完成时间 / 该公式平均时间 | 20% |
| 推导流畅度 | 推导卡自评分 + 是否需查看推导链 | 15% |
| 条件意识 | 条件 cloze 命中率（独立统计） | 15% |
| 题型识别 | 反向卡正确率 | 15% |

输出 4 级标签：**未学 → 熟悉 → 熟练 → 精通**，与 Khan Academy 一致，便于用户心理建模。

## 七、跨阶段调度与算法

### 7.1 间隔重复算法选型

**不要重新发明轮子，但要在 FSRS 之上做公式特化。**

- 基础引擎：**FSRS（Free Spaced Repetition Scheduler）**，其 DSR（Difficulty‑Stability‑Retrievability）三组件模型已开源（fsrs4anki），相比 SM‑2 减少 15‑20% 复习次数，已是 Anki 23.10+ 的可选默认。
- 备选/参考：墨墨的 DHP 模型 + SSP‑MMC（KDD 2022, IEEE TKDE 2023），针对复杂卡片设计更细。

**针对公式的三处特化**：

1. **每个公式有 N 张关联子卡（拆块 cloze + 条件卡 + 推导卡 + 辨析卡 + 题型卡）**——它们的 stability **共享一个底层"母 stability"** 但有独立的 retrievability，避免互相污染。这是 FSRS 社区讨论复杂卡片时的主流方案。
2. **数学的 spacing 效应弱于词汇**（Donovan & Radosevich 元分析）→ 间隔不能像单词那样激进拉长。建议 desired retention 默认 **90%**（FSRS 默认），冲刺期上调到 92‑95%。
3. **错误反馈不应像 SM‑2 那样直接 reset**——用 FSRS 的"重新计算 stability"，避免学生因为某天状态差而把整章公式全打回新卡。

### 7.2 一轮/二轮/三轮/冲刺推送策略

| 阶段 | 新卡上限 | 复习卡上限 | 主要题型 | 交错策略 |
|---|---|---|---|---|
| 一轮 | 6‑10/天 | 30‑50/天 | 识别 + 单 cloze | 章内 block，章间不交错 |
| 二轮 | 3‑5/天（补漏）| 60‑100/天 | cloze + 选项 + A/B 辨析 | 章内交错，跨章浅交错 |
| 三轮 | 1‑3/天 | 80‑120/天 | 默写 + 题型反查 | 全交错 |
| 冲刺 | 0 | 50‑100/天 | 闪卡识别 + 条件 + 错题专项 | 按"易遗忘度"排序，不再交错 |

### 7.3 易混公式：交错还是分散？

研究证据（Rohrer 2015；Carvalho & Goldstone 2014）支持**交错**——但有一个例外：**初次学习时不要交错（否则违反 desirable difficulty 的"learner must have background"前提）**。
- **一轮**：易混公式各自先 block 学习
- **二轮起**：易混对必须紧邻出现（A/B 辨析卡），或在同一会话窗口（相隔 2‑5 张）
- 不要让"两个易混公式"在不同日子出现——会增加 interference 风险（SuperMemo Rule 11）

### 7.4 错题反向触发该公式的复习

外部接口（与用户的纸质或电子错题本对接）：
- 用户在错题本中标记"我这道题是因为公式 X 用错"
- 系统：公式 X 的 retrievability 直接降为 0.5（无论真实状态），强制次日出现
- 增加该公式的"common_errors"计数 +1，下次出现时优先挖那个错点

## 八、数据结构与卡片设计建议

### 8.1 一个公式 → 6 类卡片的派生矩阵

| 卡片类型 | 正面 | 反面 | 主要训练 | 出现阶段 |
|---|---|---|---|---|
| **C1 识别卡** | 公式名 / 题面线索 | 完整公式 + 条件 | 形‑名映射 | 一轮起 |
| **C2 拆块 Cloze** | 公式（部分挖空）| 完整公式 + 该位口诀 | 结构化记忆、关键节点 | 一轮起 |
| **C3 条件卡** | 公式 + "本公式适用条件是？" | 条件清单 | 条件警觉，避免"标准零分" | 一轮起 |
| **C4 推导卡** | 父公式 + 目标公式名 | 推导链动画 + 用户自评推没推出来 | 程序性 + 推导流畅度 | 二轮起 |
| **C5 辨析卡** | 题面或表达式 + "选 A 还是 B"| 正解 + 区别说明 | 抗干扰、易混分离 | 二轮起 |
| **C6 题型应用卡** | 真题片段 | 用此公式的关键步骤 | 题型→公式反向激活 | 二轮起 |

每个公式至少派生 C1 + C2 + C3，三轮后把 C4/C5/C6 加入。

## 九、反模式（应该避免的设计）

借鉴 SuperMemo Rule、Bjork undesirable difficulty、不背单词差评、Anki 医学生大量踩坑经验：

1. **❌ 直接照搬"四次连续答对"模式**——不背单词只要"识别+拼写"四次过就掌握，但公式答对不等于会用，必须四次过 ≠ 同一类型四次，应该是"识别 + 拆块 + 条件 + 题型"四种不同维度各一次。

2. **❌ 把"识别选 A/B/C/D 选项"作为主要复习手段**——recognition 不等于 recall（Bjork、levels‑of‑processing）。选项题只能作为最低难度档，二轮后比例必须降到 <30%。

3. **❌ 过早或过频繁默写**——novice 阶段强制默写违反 worked‑example effect（Sweller），会让用户产生畏难情绪而流失。一轮以识别+ cloze 为主，二轮才大量默写。

4. **❌ 把整条长公式作为单一 cloze 卡**——SuperMemo Rule 4（minimum information）和 Chase‑Simon chunk size 4‑7 都警告我们：长公式必须拆块。否则 FSRS difficulty 会一直爆表，用户卡在"ease hell"。

5. **❌ 列举型卡（"列出所有等价无穷小"）**——SuperMemo Rule 9/10 明确反对 sets。必须改成"sin x ~ ?"逐条 cloze，或图谱化。

6. **❌ 像 Duolingo 那样过度 streak/guilt**——考研用户压力极大，"一天没打卡 streak 清零"会触发焦虑放弃。建议改为**周打卡 + 复习覆盖率**（墨墨"记忆持久度"模式）。

7. **❌ 让用户自己决定"我学会了"（高 JoL）**——Koriat & Bjork foresight bias 和"easily learned, easily remembered"启发式都表明用户会高估自己。必须以**测试结果**而非用户自评作为掌握判定（这是 FSRS 而非 SM‑2 的优势：你按"Easy"不直接放飞，模型仍会校准）。

8. **❌ 把"不背单词"的 listening / 真实语境例句直接搬过来**——数学没有"语境例句"那样的真实场景；强行搬只会让卡片臃肿。**要替换为：题面 + 易错点 + 推导链**——这才是公式的"语境"。

9. **❌ 缺少"易混对"的显式辨析**——很多 Anki 卡组失败原因就是 interference 没处理。Formula Master 必须把易混对当一等公民。

10. **❌ 完全靠 OCR/手写识别判定对错**——目前的手写识别（MyScript、Mathpix Digital Ink、Selvy Pen SDK）对手写公式识别率约 90‑95%，会有误判挫败用户。建议手写后让用户**自评（对/小错/不会）**，识别结果仅作辅助显示，避免 false negative 污染 FSRS。

## Caveats / 局限说明

1. **认知科学结论的边界**：交错练习、testing effect 等的效应量主要在西方 K‑12 受试者上验证，对中国成年考研学生的迁移性虽然有大量间接证据（中国教育研究、墨墨用户行为分析），但尚无针对"考研数学公式"的直接 RCT。建议产品上线后做 A/B 实验自行验证。
2. **数学的 spacing 效应可能弱于词汇**：Donovan & Radosevich (1999) 元分析指出 spacing 效应随任务复杂度上升而减弱。这意味着 FSRS 的默认参数可能需要针对数学公式重新调优——建议在用户量积累 1000+ 后用 FSRS Optimizer 重新拟合。
3. **手写识别误判**：所有手写公式 SDK 在复杂表达式（多重积分、矩阵）上仍有 5‑10% 错误率。前期不要把识别结果作为"对/错"硬判定。
4. **名师方法的产品化局限**：汤家凤的板书风、张宇的口诀、武忠祥的"经典错误"都依赖讲师在场的语言艺术，App 内只能以视频/图文/动画**部分还原**，需要内容团队精心改编。
5. **关于"5 秒推导"**：仅适用于推导链短的公式族（诱导、和差、半角等），对于推导冗长（如分部积分一般情形、二项式定理一般证明）仍然要直接背结论。
6. **不背单词的"四次答对"机制的引用**：来源于民间产品分析与汉斯出版社论文，不背单词官方未公开发布算法细节；具体次数与触发逻辑可能随版本变化。
7. **墨墨 DHP/SSP 算法的开源版**为 FSRS（同一作者 Jarrett Ye），生产环境部署 FSRS 比直接抄墨墨更稳妥。
8. **"知能行"App 的实证**目前主要见于知乎用户口碑，未见严格学术评估。
9. **SuperMemo "20 rules"**是工程实践经验法则，并非全部都有 RCT 支持，但在 SR 社区有 25 年实战检验，可作为强先验使用。
10. **本报告未涵盖的方向**（建议后续补充研究）：数学公式 OCR 自动从教材/讲义生成 Formula Object 的可行性；AI（LLM）辅助生成"个性化推导动画"和"个性化口诀"的产品价值；长期使用带来的依赖性与"Anki 倦怠"问题的对策（参考 Reddit r/Anki 大量讨论）。
