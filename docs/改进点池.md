### [交互] 全面震动 / 触觉反馈适配
- **优先级**：P2
- **日期**：2026-05-01
- **背景**：当前震动反馈仅在严测/复习的"出错"场景触发（[`vibrateError`] 200/400ms），其余交互
  全部静默。但震动作为"已操作成功"的体感锚点，能显著降低用户对界面响应性的怀疑。
- **建议覆盖范围**：
  - **复习评分按钮（1/2/3/4）点击**：短促 confirm 震动（HapticFeedback.LongPress 等级）
  - **严测「都不对」/「提交核对」按钮**：confirm
  - **公式激活（首次进入复习队列）**：success 震动 + 可视化反馈
  - **设置页保存 / 测试连接成功**：success
  - **Onboarding 页面切换 / 完成「开始使用」**：subtle tick
  - **Pager 翻页边界（已是第一/最后一页还在拖）**：edge tick
  - **错误场景扩展**：识别失败 / 网络超时 / Snackbar error 弹出时同步震动（与 vibrateError 共用 API）
- **实现思路**：
  1. 抽出 `ui/util/Haptic.kt`（LocalHapticFeedback 包装 + 兜底 Vibrator API），区分 5 档：
     `Tick` / `Confirm` / `Success` / `Warning` / `Error`
  2. 各 Composable 通过 `LocalHapticFeedback` 取实例后按场景调用
  3. 设置页加"震动反馈"开关（默认 ON），用户可一键全局关闭
- **关联文件**：`ui/util/Haptic.kt`（新建）、`ui/component/TestCanvas.kt`、`ui/component/ReviewCard.kt`、
  `ui/screen/SettingsScreen.kt`、`ui/screen/OnboardingScreen.kt`、`data/AppPreference.kt`

### [工程] 日期时间库迁移至 kotlinx-datetime（KMP 扩展准备）
- **优先级**：P3
- **日期**：2026-04-30
- **描述**：当前 domain 层（`ReviewScheduler.truncateToRefreshHour` 等）使用 `java.time`
  （`ZonedDateTime` / `Instant` / `ZoneId`）。`java.time` 在 minSdk ≥ 26 时原生可用，是现阶段最优选。
  若未来将 domain 层扩展为 Kotlin Multiplatform（iOS / Desktop），需切换为
  `kotlinx-datetime`（JetBrains 官方 KMP 日期库），因为 `java.time` 不跨平台。
- **迁移成本估计**：低。`kotlinx-datetime` API 与 `java.time` 高度对应
  （`Clock.System.now()` → `Instant.now()`，`TimeZone.of()` → `ZoneId.of()`，
  `LocalDate` / `LocalDateTime` 同名），主要是 import 替换 + 少量 API 差异调整。
- **重启条件**：项目启动 KMP 改造（共享 domain 层到非 Android 平台）时纳入。
- **关联文件**：`domain/ReviewScheduler.kt`（当前唯一用到 `java.time` 的 domain 文件）

### [性能] 复习/详情页 KaTeX 渲染时长进一步优化

### [识别] 端侧 TFLite/ONNX 手写公式识别（L1 本地方案，聚焦考研符号集）
- **优先级**：P3
- **日期**：2026-04-25
- **描述**：ML Kit 不支持数学公式识别，但以下端侧路线技术上完全可行，且比通用数学识别更聪明：
  - **输入**：笔画时序数据 `[(x,y,t)]`，比图像 OCR 轻量，无需拍照/截图
  - **模型**：小型 LSTM 或 TCN，针对考研数学核心符号集（~150个：高数、线代、概率的常见符号和结构），
    基于 CROHME 数据集（Competition on Recognition of Online Handwritten Mathematical Expressions）训练
  - **部署**：转换为 TFLite 或 ONNX Runtime Mobile 格式，模型大小目标 10~30MB，完全离线运行
  - **识别范围**：考研数学常见符号和简单嵌套结构（如 `\int_a^b f(x)dx`、`\frac{d}{dx}`、行列式等）；
    复杂多级嵌套（如三重积分）识别率会下降，但覆盖大多数考研题型
  - **工程步骤**：处理 CROHME InkML 数据 → 训练符号分类器 → 结构解析层（上下标、分式等） → TFLite 转换 → Android 集成
- **关联文件**：待新建 `domain/TfLiteInkRecognizer.kt`；需新增 TFLite 依赖
- **主要挑战**：
  1. 目前无现成可用的 TFLite 数学公式识别模型，需自行训练（约 1-2 周，GPU 可用 Colab 免费额度）
  2. 符号识别 → LaTeX 组装需要结构解析层，工程量常被低估
  3. 本质是之前搁置的 L2 的精简版，门槛仍需要 ML 训练能力
- **启动条件**：A1/A2 API 路线稳定运行后，有意愿投入 ML 训练时再做
