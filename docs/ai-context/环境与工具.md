## Android CLI（`android` 命令，本机已装 · PATH 直达）

**优先用它替代手敲 `./gradlew installDebug` + adb。**

| 子命令 | 用途 | 典型场景 |
|--------|------|---------|
| `android run` | 编译+安装+启动到设备/模拟器 | 替代 `gradlew installDebug && adb shell am start` |
| `android emulator` | 列/启/关 AVD | 纸笔横屏起 tablet AVD；Onboarding 冷启动 wipe data |
| `android layout` | 抓当前 App 布局树（Compose 友好）| UI bug 排查——看节点尺寸/层级，替代部分诊断角标 |
| `android screen` | 截屏/录屏 | 自己抓 UI 反馈，减少用户手动截图回合 |
| `android describe` | 项目元数据摘要（manifest/gradle）| 给新对话快速喂背景 |
| `android sdk` | 列/装 SDK 组件 | 装平台/build-tools/system-image |
| `android docs` | 离线查 Android API 文档 | 替代 WebFetch developer.android.com |
| `android info` | SDK Location/环境信息 | 排查 KSP/AGP 版本冲突 |

**注意**：① 网络隔离环境首次运行可能下载内嵌资源需用户确认；② 默认收匿名 telemetry，加 `--no-metrics` 关；
③ **第三方 API（Mathpix/SimpleTex 等）能力/定价仍必须 WebFetch 官方页**，`android docs` 只覆盖 Android 平台 API（见协作约定 B5）。

## 真机 DB 调试：pull-modify-push 标准流程

Android 设备常**不带 sqlite3 二进制**，不能在设备上直接改库。真机回归需构造特定状态（"只让某子卡 due"/"模拟 blocked"）时走这套：

```bash
ADB=~/Android/Sdk/platform-tools/adb
PKG=com.example.formulamaster

# 1. 停 app 让 SQLite 把 WAL 落盘
$ADB shell am force-stop $PKG; sleep 1

# 2. Pull 三件套（真实数据在 .db-wal，只拉 .db 会得到空壳！）
$ADB exec-out run-as $PKG cat databases/formula_master.db     > /tmp/fm.db
$ADB exec-out run-as $PKG cat databases/formula_master.db-wal > /tmp/fm.db-wal
$ADB exec-out run-as $PKG cat databases/formula_master.db-shm > /tmp/fm.db-shm

# 3. 本地 python3 改（设备无 sqlite3，但 python 几乎都有）
python3 - <<'EOF'
import sqlite3
conn = sqlite3.connect('/tmp/fm.db')
conn.execute("PRAGMA wal_checkpoint(TRUNCATE);")  # 关键：WAL 合并进主库
# ... UPDATE / DELETE / SELECT ...
conn.commit(); conn.close()
EOF

# 4. 推回（run-as 读不了 /sdcard，走 /data/local/tmp 中转；先删旧 WAL/SHM）
$ADB push /tmp/fm.db /data/local/tmp/fm.db
$ADB shell run-as $PKG rm -f databases/formula_master.db-wal databases/formula_master.db-shm
$ADB shell run-as $PKG cp /data/local/tmp/fm.db databases/formula_master.db
$ADB shell rm /data/local/tmp/fm.db

# 5. 重启
$ADB shell am start -n $PKG/.MainActivity
```

**踩过的坑**：① `adb shell sqlite3` → 设备无此二进制；② 只 pull `.db` → 4KB 空壳无数据；③ 推 sdcard 再 cp → run-as 无权读 /sdcard；④ 不删 WAL 直接 cp → SQLite 检测主库 page count 与 WAL 不匹配，可能崩或用旧 WAL 覆盖你的改动；⑤ 跨 destructiveMigration 升级后旧 dump 失效，需重新 pull。

**无线调试**：`adb connect <手机IP>:<端口>`（端口每次变；VPN/Clash TUN 模式会给假 IP 198.18.x.x 导致配对失败，需先关 VPN）。
