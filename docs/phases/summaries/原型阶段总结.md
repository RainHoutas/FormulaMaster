## 二、核心技术决策

| 主题 | 决策 | 原因 |
|------|------|------|
| KSP 版本 | `2.2.10-2.0.2` | 匹配 Kotlin 2.2；旧式 1.0.x 不兼容 AGP 9.x |
| AGP 9.x 插件 | 不显式声明 `kotlin.android` | 已 built-in，显式声明会冲突 |
| Kotlin sourceSets | `android.disallowKotlinSourceSets=false` | AGP 9 默认收紧规则，老代码布局需兜底 |
| 公式渲染 | KaTeX 离线包 + WebView | 禁止 JLatexMath（Compose 原生 Text 无法还原分式/矩阵） |
| WebView 深色 | `body.dark` CSS + Kotlin 注入 | 各厂商 ROM `prefers-color-scheme` 实现不一致 |
| WebView BaseURL | `loadDataWithBaseURL("file:///android_asset/", …)` | 否则 KaTeX JS/CSS/字体被 WebView 安全策略拦截 |
| FSRS 遗忘惩罚 | 强制 `learningState=1` + S 清零 + `lapses++` + `consecutiveGoodReviews=0` | 测试模式 `rating=1` 为最严判决 |
| FSRS 测试奖励 | R=4 奖励 ×1.5 稳定度 | 默写对比选项难度更高 |
| Tab 切换动画 | 全局 `EnterTransition.None`，子路由各自声明 `slide` | 避免 Tab 切换闪烁，详情页独立滑入 |
| 复习队列冻结 | ViewModel 首次加载冻结 `sessionItems` | 答题落库立即会让 HorizontalPager 跳页 |
| 状态栏 inset | 每屏自管（`MainScreen` 不给 NavHost 加 padding） | 详情页/测试页有独立 Scaffold，避免双重消费 |
| SprintMode 幂等 | `halveStabilityAbove(15.0)` | 避免多次进入冲刺反复折半 |
| Test 路由 | 从 `topLevelRoutes` 移除 | 进入测试隐藏底栏，营造考试氛围 |
| WebViewPool 容量 | 3 | 同屏最多 3 个公式（列表/复习/候选），够用不浪费 |
| 通知调度 | `PeriodicWorkRequest` + `KEEP` + 对齐 08:00 | 重复调度不覆盖、时间可预测 |
| Heatmap 宽度 | 固定 `WEEKS=53` + 外层 `horizontalScroll` | 保持格子 12dp 辨识度，溢出留给用户横滑 |

## 四、已知 UX 缺陷（供 Phase 2 参考）

- **HeatmapCalendar 宽度溢出**：53 列 × (12+3)dp ≈ 795dp，需手动横滑；可考虑自适应缩放或按月分段
- **OCR 候选始终假数据**：Mock 实现不解析 strokes，无法体现真实默写体验
- **公式库过小**：6 条种子数据，无法支撑真实复习节奏与 FSRS 曲线观察
- **主题色仅蓝色 Fallback**：未做数学/学术风格定制
- **TestScreen 只有手写**：软键盘输入 / 结构化编辑器未实现，手写识别未落地前答题门槛偏高
- **冲刺模式无 UI 提示**：距考试 30 天内首页无倒计时 Banner，用户不易感知模式切换
- **通知小图标**：直接复用 launcher foreground，在通知栏可能呈现为白色方块（缺 monochrome）

## 七、注意事项（防踩坑）

1. WebView 只能主线程操作：`WebViewPool.acquire/release` 必须在主线程
2. Room WAL 模式：调试需同时拷贝 `.db + .db-wal + .db-shm`，或用 App Inspection
3. `material-icons-core` 仅含基础图标集，`extended` 未引入；`Undo/DeleteSweep` 等用 `AutoMirrored.ArrowBack + Delete/Clear` 代替
4. `acquire(context)` 传 Activity context 即可，Pool 内部不持有 Activity 引用
5. Scaffold 的 `bottomBar` 若非 `BottomAppBar`（例如直接塞 `FilledTonalButton`），需自行 `Modifier.windowInsetsPadding(WindowInsets.navigationBars)`，否则手势导航栏机型会遮挡
6. `SprintModeManager.halveStabilityAbove(15.0)` 自带幂等保护：稳定度降至 ≤15 后不会被再次减半
